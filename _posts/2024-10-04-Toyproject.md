# Toy Project

## XAI (Explainable Artificial Intelligence)

XAI refers to a set of processes and methods that allow users to understand and trust the results and outputs generated by machine learning algorithms.

- XAI helps explain AI models, their expected impacts, and potential biases.
    
    → Assists in characterizing model accuracy, fairness, transparency, and outcomes in AI decision-making.
    
    → Plays a critical role in building trust when deploying AI models into production.
    
    → Explainability through AI helps organizations adopt responsible approaches to AI development.

Typically, the entire calculation process changes into an entity called a **black box**, which is usually difficult to interpret. (Black box models are generated directly from data.)

- Explainability helps developers verify whether the system is functioning as expected.
- It is essential for meeting regulatory standards or enabling those affected by decisions to challenge or alter the outcomes.

## CAM (Class Activation Mapping)

CAM highlights the discriminative image regions used by a CNN when classifying an image.

![img1.daumcdn.png](/assets/images/post_data/Toyproject_image/img1.daumcdn.png)

It mainly consists of convolution layers, with GAP (Global Average Pooling) performed before the final output layer (classification: softmax), and CAM generated using the features of the FC (fully connected) layer.

The weights of the output layer are projected back onto the class activation mapping to generate CAM.

GAP outputs the spatial average of each unit’s feature map from the last convolutional layer.

→ The weighted sum of these values is used to produce the final output.

→ CAM is obtained by calculating the weighted sum of the feature maps from the last convolution layer.

For a given image, \( f_k(x,y) \) represents the activation of unit \( k \) (the \( k \)-th feature map: \( f_k \), where \( f_k(x, y) \) represents the values of the \( k \)-th feature map at coordinates \( (x, y) \)).

The result of performing GAP on \( f_k \) is \( F^k \).

![img1.daumcdn.png](/assets/images/post_data/Toyproject_image/img1.daumcdn%201.png)

Given a class \( c \), the softmax input is \( S_c \).

\( w \) is the weight of class \( c \) for \( f_k \) (the importance of \( F_k \) in class \( c \)).

model

```python
base_model = models.vgg19(weights=models.VGG19_Weights.IMAGENET1K_V1)
base_model.classifier = nn.Identity()

class VGG19_with_GAP(nn.Module):
    def __init__(self):
        super(VGG19_with_GAP, self).__init__()
        self.base = base_model.features
        self.gap = nn.AvgPool2d(7)
        self.fc = nn.Linear(512, 10)
        self.softmax = nn.Softmax(dim=1)

        for param in list(self.base.parameters())[:-4]:
            param.requires_grad = False

    def forward(self, x):
        x = self.base(x)
        features = x.clone()
        x = self.gap(x)
        x = x.view(x.size(0), -1)
        x = self.fc(x)
        x = self.softmax(x)
        return x, features

pretrained_model = VGG19_with_GAP()
```

```python
criterion = nn.CrossEntropyLoss().cuda()
optimizer = optim.Adam(pretrained_model.parameters(),lr=0.00001)

device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
pretrained_model.to(device)

for epoch in range(num_epochs):
    pretrained_model.train()
    running_loss = 0.0
    running_corrects = 0
    for inputs, labels in train_loader:
        inputs, labels = inputs.to(device), labels.to(device)

        optimizer.zero_grad()

        outputs, _ = pretrained_model(inputs)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()

        _, preds = torch.max(outputs, 1)
        running_loss += loss.item() * inputs.size(0)
        running_corrects += torch.sum(preds == labels.data)

    epoch_loss = running_loss / len(train_loader.dataset)
    epoch_acc = running_corrects.double() / len(train_loader.dataset) * 100

    print(f'Epoch {str(epoch+1).zfill(2)}/{num_epochs}, Loss: {epoch_loss:.4f}, Accuracy: {epoch_acc:.4f} %')
```
